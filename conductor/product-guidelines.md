# Product Guidelines: YouTube to Guides

## Voice and Tone

**Concise and direct.**

- CLI output should be brief and informative — no fluff
- Error messages should clearly state what went wrong and how to fix it
- Progress indicators should be minimal but informative
- Generated guide content should mirror the source material's clarity

## Design Principles

**Developer experience focused.**

- Simple, intuitive CLI commands with sensible defaults
- Clear error messages with actionable guidance
- Fast feedback loops — show progress during long operations (video download, AI processing)
- Minimal configuration required to get started
- Outputs should be immediately usable without post-processing

## Output Standards

- Generated guides use Markdown as the primary format (portable, readable, convertible)
- Images embedded with descriptive alt text
- Sections and headings follow a logical hierarchy
- Steps are numbered and self-contained
