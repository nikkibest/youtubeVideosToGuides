# Python Style Guide: YouTube to Guides

## Formatting

- Use `ruff format` for auto-formatting (replaces Black)
- Line length: **88 characters**
- Use double quotes for strings

## Linting

- Use `ruff check` for linting
- Follow PEP 8 conventions
- No unused imports — remove them immediately

## Naming Conventions

- `snake_case` for functions, variables, and modules
- `PascalCase` for classes
- `UPPER_SNAKE_CASE` for constants
- Prefix private functions/methods with `_`

## Type Hints

- Use type hints on all public functions
- Use `from __future__ import annotations` for forward references
- Prefer `X | Y` over `Union[X, Y]` (Python 3.10+)
- Use `list[str]` not `List[str]`, `dict[str, int]` not `Dict[str, int]`

```python
def extract_transcript(url: str, language: str = "en") -> list[str]:
    ...
```

## Functions

- Keep functions small and focused — one responsibility
- Max ~30 lines per function; split if longer
- Use descriptive names — avoid abbreviations

## Error Handling

- Use specific exceptions, not bare `except:`
- Raise early, return results at the end
- Use custom exception classes for domain errors

```python
class TranscriptNotAvailable(Exception):
    pass
```

## Imports

Order: stdlib → third-party → local, separated by blank lines

```python
import os
from pathlib import Path

import typer
from anthropic import Anthropic

from .transcript import extract_transcript
```

## CLI (Typer)

- Use `typer.Typer()` app instance
- Annotate CLI arguments and options with type hints and `typer.Argument` / `typer.Option`
- Provide `help=` strings for all arguments and options
