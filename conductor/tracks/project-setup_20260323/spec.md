## Summary

Set up the Python project foundation including package structure, dependencies, CLI entry point, and dev tooling so the codebase is ready for feature development.

## Context

YouTube to Guides is a CLI tool that downloads YouTube videos, extracts transcripts and frames, and uses Claude AI to produce structured, step-by-step Markdown guides. This track lays the groundwork — without it, no other feature can be built.

## User Story

As a developer/user, I want a clean Python package structure and all key dependencies installed so I can load a YouTube video by URL, extract its data, and feed it into a guide generation pipeline.

## Acceptance Criteria

- [ ] Project installs cleanly with `uv` (or `pip`) from a `pyproject.toml`
- [ ] Running `ytguide --help` prints usage info
- [ ] Running `ytguide generate <youtube-url>` fetches video metadata and transcript without error
- [ ] Video frames/screenshots can be extracted from a downloaded video
- [ ] Output is consistent in style across different videos (same Markdown template applied)
- [ ] `ruff` passes with no errors

## Technical Notes

- Use `uv` for package/environment management (`pyproject.toml` + `uv.lock`)
- CLI via `typer` + `rich` for progress/output
- `yt-dlp` for video download and metadata; `youtube-transcript-api` for transcripts
- `opencv-python` for frame extraction; `Pillow` for image handling
- `anthropic` SDK included as dependency (wired up in a later track)
- Consistent output style enforced via a shared Markdown template module
